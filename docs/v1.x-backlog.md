**Surfaced in:** P28 sign-off run.
**Why deferred:** Demo Co's full-day walkthrough needed HR /

   + a Settings → Users page in the SPA. Email invitation flow

**Files to touch:** `backend/scripts/`, new
`backend/maugood/users/router.py`, frontend Settings nav.

### B-2 — Background-job entry points use TenantScope default schema (medium)

**Surfaced in:** P28 sign-off run.
**Why deferred:** `TenantScope(tenant_id=N)` defaults
`tenant_schema='main'`. Background-job entry points
(`recompute_today`, `report_runner._tick`,
`notification_worker._tick`, etc.) call
`tenant_context(scope.tenant_schema)` with `"main"` regardless
of the real tenant. Symptom: jobs silently process tenant 1's
data when invoked with another tenant's `TenantScope`.

**Fix shape:**
1. Add `TenantScope.for_tenant(engine, tenant_id)` classmethod
   that resolves the schema via
   `resolve_tenant_schema_via_engine`.
2. Audit every `TenantScope(tenant_id=...)` call site outside
   the request path; switch to the new classmethod.

**Files to touch:** `backend/maugood/tenants/scope.py`,
`backend/maugood/attendance/scheduler.py`,
`backend/maugood/scheduled_reports/runner.py`,
`backend/maugood/notifications/worker.py`,
`backend/maugood/retention/sweep.py`.

### B-3 — Runbook `.env` placeholder syntax is ambiguous (low)

**Surfaced in:** P28 sign-off run.
**Why deferred:** `docs/deploy-production.md §3` shows literal
`<MAUGOOD_APP_DB_PASSWORD>` inside `MAUGOOD_DATABASE_URL` — a
careless paste-and-edit operator could leave the angle-
brackets in.

**Fix shape:** add a one-line note that the placeholder must
be substituted with the literal password value, or use
shell-friendly templated form
(`${MAUGOOD_APP_DB_PASSWORD}`) and explain the substitution.

**Files to touch:** `docs/deploy-production.md`.

### B-4 — email-validator rejects .local TLD (low)

**Surfaced in:** P28 sign-off run.
**Why deferred:** the runbook recommends `maugood.local` for the
mkcert dev recipe; an operator using the same domain for emails
(`admin@maugood.local`) gets a 422 from `email-validator`. Real
public TLDs work fine.

**Fix shape:** add a callout in the mkcert recipe that emails
need a public TLD (`admin@example.com`) even when the hostname
doesn't.

**Files to touch:** `docs/deploy-production.md §10`.

### B-5 — Backend image debian-trixie OS CVEs (medium, no fix yet)

**Surfaced in:** P27 security review.
**Why deferred:** 142 HIGH OS CVEs from `python:3.11-slim`
base, all `fixed-avail=0` upstream. Mitigation: monthly
rebuild as Debian publishes patches.

**Fix shape:** schedule a monthly CI job that rebuilds
`maugood-backend:latest` from the latest base, runs `pytest -q`,
and ships if green. A `chore(deps): refresh base images`
commit per month.

**Files to touch:** `.github/workflows/` (new monthly job).

### B-6 — gosu CVEs in postgres-alpine base (medium)

**Surfaced in:** P27 security review.
**Why deferred:** 8 HIGH + 1 CRITICAL CVEs in `gosu` shipped by
`postgres:15-alpine` base on the backup image. Maugood's backup
script doesn't invoke gosu (we run as root inside the
short-lived container) so the binary is dormant. Replacing
requires rebasing on a different image.

**Fix shape:** swap the backup image base to a slim postgres-
client distribution that doesn't ship gosu (e.g., a
custom `python:3.12-slim` with `apt install postgresql-client`).
Re-validate with `tests/test_p21_i18n_lint.py`-style smoke.

**Files to touch:** `ops/backup/Dockerfile`.

### B-8 — Fernet-key rotation tooling (medium)

**Surfaced in:** P27 security review.
**Why deferred:** `MAUGOOD_FERNET_KEY` rotation today is
destructive — the new key can't decrypt the old ciphertext.
Annual rotation is documented but not automated.

**Fix shape:** support a two-key window (current + retiring)
+ a re-encrypt background job that walks every encrypted
asset (`employee_photos`, `cameras.rtsp_url_encrypted`,
`request_attachments`, `email_config.*_password_encrypted`,
`tenant_oidc_config.client_secret_encrypted`) and rewrites
ciphertext under the new key. After the job completes, drop
the retiring key.

**Files to touch:** new `backend/maugood/crypto/rotation.py`,
ops runbook.
