# P15 — Approvals inbox

**Phase:** v1.0 P15 (M2)
**Date (UTC):** 2026-04-25
**Engineer:** Suresh (MTS) via Claude Code
**Plan reference:** `v1.0-phase-plan.md §"P15 — Approvals inbox"`

## Goal

Replace the pilot Approvals placeholder with a working inbox for
Manager / HR / Admin. Three tabs (Pending mine / Decided by me /
All — Admin only). Per-row metadata for attachments, days open
(business hours, tenant weekends honoured), and SLA breach
indicator. Sidebar badge driven by a live "pending mine" count.

## What was done

### Backend

* **Widened manager scope.** Previously the manager could only see
  + decide on requests for employees with an explicit
  `manager_assignments` row. P15 unions that with
  `user_departments` membership (the same set
  `get_manager_visible_employee_ids` from P8 surfaces for the
  attendance scope). Both `_can_view` and `manager_decide_endpoint`
  use the wider set; explicit assignments still drive the
  primary-manager auto-routing on submission. The change is
  load-bearing for the P15 ask "directly assigned OR in their
  department(s)".
* **New repository helpers** (`maugood/requests/repository.py`):
  * `list_pending_manager_for_employees(...)` — submitted requests
    for a manager's visible employees.
  * `list_pending_hr(...)` — manager-approved requests across the
    tenant.
  * `list_decided_by_user(...)` — every request the user has
    touched at any decision stage.
  * `primary_managed_employee_ids(...)` — subset of visible
    employees flagged `is_primary` (used to sort pending and to
    surface the badge in the UI).
* **SLA module** (`maugood/requests/sla.py`) — pure business-hours
  computation. Per-day pro-ration, weekend-aware via the tenant's
  `tenant_settings.weekend_days` (P11). Two functions:
  `business_hours_open(submitted_at, as_of, config)` and
  `is_breached(...)` for the threshold check.
* **Two new config knobs** in `maugood/config.py`:
  `request_sla_business_hours` (default 48) and
  `request_sla_business_day_hours` (default 8). Tunable per tenant
  by env override (BRD Open Item Q6).
* **Inbox endpoints** (router static-paths declared **before**
  `/{request_id}` so FastAPI's matcher picks them first):

| Method + Path | Notes |
| --- | --- |
| `GET /api/requests/inbox/pending` | Role-scoped "Pending my decision". Manager sees submitted-on-visible employees, primary-assigned first; HR sees `manager_approved`; Admin sees every non-terminal. |
| `GET /api/requests/inbox/decided` | Every request the caller has touched at any stage (manager / HR / admin). |
| `GET /api/requests/inbox/summary` | Sidebar badge feed: `{pending_count, breached_count}`. |

* **Response enrichment.** `GET /api/requests`, the inbox endpoints,
  and every single-row endpoint (`get_request`, `cancel`,
  `manager_decide`, `hr_decide`, `admin_override`,
  `create_request`) all run through `_enrich_responses` /
  `_enrich_single`, which join attachment counts in one bulk SQL
  and compute `business_hours_open` + `sla_breached` +
  `is_primary_for_viewer` per row.

### Frontend

* **Approvals page** at `/approvals`
  (`frontend/src/requests/ApprovalsPage.tsx`). Three tabs (Pending
  mine / Decided by me / All — Admin only). Table columns:
  Employee (with primary badge when applicable), Type, Reason,
  Date(s), Days open (business hours; warns past threshold),
  Status + Stage label, Files count.
* **Detail drawer extension**
  (`frontend/src/requests/RequestDetailDrawer.tsx`) — gains an
  optional `decisionRole` prop. When set, the foot renders a
  `DecisionFooter` with a comment textarea and Approve / Reject
  buttons, gated per role:
  * Manager — only when `status=submitted`; comment optional on
    approve, mandatory on reject.
  * HR — only when `status=manager_approved`; same rule.
  * Admin — universally allowed; comment **mandatory** (BRD
    FR-REQ-006). Backend's 422 on empty / whitespace comment is
    mirrored client-side.
* **Sidebar badge live count.** `Sidebar.tsx` calls
  `useInboxSummary()` for Manager/HR/Admin and overrides the
  static design-archive badge on the Approvals nav item with the
  real `pending_count` (or hides it at 0). When
  `breached_count > 0` the badge takes a danger tone so an
  operator who's drifted past the SLA on their queue sees it
  immediately.
* **TanStack Query invalidation.** Decision hooks
  (`useManagerDecide` / `useHrDecide` / `useAdminOverride`)
  invalidate `["requests"]`, the three inbox keys, and the
  summary key on success. The summary query also opts into
  `refetchOnWindowFocus` so a tab switch refreshes the badge
  without manual reload.

### Routing

`App.tsx` registers `/approvals → ApprovalsPage` and removes
`approvals` from the placeholder filter. The new route is
backed by the existing `approvals` nav id (already in
`shell/nav.ts` for Manager / HR / Admin).

## Notes for whoever resumes next

- **Admin override UI (P16)** is now mostly done — the detail
  drawer's `DecisionFooter` already handles `decisionRole="Admin"`
  with the mandatory-comment guard. P16 just needs to expose the
  Admin override entry point on terminal rows (today only the
  Admin tab/Pending tab opens the drawer with the decision
  footer; on a terminal row the footer hides). One small follow-up
  is to allow the admin to open *any* request from the All tab
  with override enabled.
- **SLA threshold tuning.** BRD Open Item Q6 left the threshold
  for operators to dial in. The default is 48 business hours; the
  per-day cap is 8 hours. If a tenant runs 6-day weeks (no Friday
  weekend), the env override `MAUGOOD_REQUEST_SLA_BUSINESS_HOURS`
  alone won't be enough — the per-tenant
  `tenant_settings.weekend_days` already drives the calendar
  walk; a future phase could surface an per-tenant override of the
  threshold itself.
- **Sidebar badge polling.** Today the badge query is staleTime
  15s + refetch-on-focus; manager invalidations push immediately.
  That's lighter than a websocket but still good enough for the
  pilot's "within seconds" contract. If it gets noisy at scale
  (lots of HR users on screens all day) flip to a longer
  staleTime + a single dashboard-wide poll.
- **`.test` TLD gotcha.** Same as P11/P12/P13/P14 — Pydantic
  EmailStr rejects reserved TLDs, so the smoke uses `.maugood`.
- **Decision red lines** are repeated server-side in the state
  machine and reinforced client-side in the DecisionFooter:
  Manager → only `submitted`; HR → only `manager_approved`;
  Admin → any state but a non-empty comment is mandatory. Even
  if a future UI bug surfaces an HR-decide on a manager-rejected
  row, the backend returns 409 with the exact reason.
