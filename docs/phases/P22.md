## Goal

Wire the dark theme + density toggles the design CSS already
shipped (`[data-theme="dark"]`, `[data-density="compact"]`,
`--density` custom property), persist the operator's choice to
their user row, surface a topbar control, and stand up the two
deferred reference pages — "How it works" and "API Reference".
Land the basic accessibility polish on every new interactive
element while we're in here.

## What was done

### Migration 0023

`alembic/versions/0023_users_theme_density.py`. Mirrors the P21
shape with two nullable `TEXT` columns on `users`:

* `preferred_theme` — CHECK locked to
  `('system','light','dark')`.
* `preferred_density` — CHECK locked to
  `('compact','comfortable')`.

`NULL` on theme means "follow OS via prefers-color-scheme";
`NULL` on density falls back to `comfortable` (the design's
default). Schema-agnostic: ran cleanly under `main`,
`tenant_omran`, and `tenant_demo`.

### `db.py`

Two `Column` declarations + two `CheckConstraint` rows added to
the `users` table (between `preferred_language` and `created_at`).

### Backend auth surface

* `CurrentUser` (`maugood/auth/dependencies.py`) gains
  `preferred_theme: str | None` + `preferred_density: str | None`.
  `_load_current_user_bundle` reads both columns and propagates
  them; the session-resolver `current_user` constructor passes
  them through every refresh path.
* `MeResponse` (`maugood/auth/router.py`) gains the matching
  fields. The login response, `/api/auth/me`, the role-switch
  refresh, and every PATCH handler now thread them.
* `PATCH /api/auth/preferred-theme` — validates against
  `('system','light','dark')`, refuses synthetic Super-Admin
  (no real users row), audits as `auth.preferred_theme.updated`.
* `PATCH /api/auth/preferred-density` — same shape, locked to
  `('compact','comfortable')`, audits as
  `auth.preferred_density.updated`.

### Frontend `theme/`

`frontend/src/theme/index.ts` is a tiny external store:

* Reads localStorage on first import (`maugood-theme`,
  `maugood-density`); falls back to `system` / `comfortable`.
* `effectiveTheme(theme)` resolves `system` against the OS
  `matchMedia('(prefers-color-scheme: dark)')`, returning
  `'light' | 'dark'` for the `data-theme` attribute. The result
  is what gets stamped — CSS rules don't have to know about
  `system`.
* `applyToRoot` sets both attributes on
  `document.documentElement`. Re-runs whenever the OS scheme
  flips while `system` is the active theme.
* `setTheme(theme)` / `setDensity(density)` — local + emit +
  PATCH the server. Network errors are swallowed (the local
  choice still applies until the user signs in elsewhere).
* `applyServerPreferences({theme, density})` — apply the values
  ride-along on `/api/auth/me` without a fresh fetch. AuthProvider
  calls this once `useMe` resolves so a fresh login on another
  browser immediately reflects the saved choice.
* `subscribe(fn)` + `getTheme` / `getDensity` — the trio
  `useSyncExternalStore` consumes from the topbar.

`main.tsx` imports `./theme` so the boot pass runs before any
component mounts (first paint already matches the saved choice).

### Topbar `DisplaySwitcher`

`frontend/src/shell/DisplaySwitcher.tsx`. Single icon button
(sun for light, moon for dark) opens a 240-px popover with two
segmented controls — Theme (System / Light / Dark) and Density
(Compact / Comfortable). Mounted between `NotificationBell` and
`LanguageSwitcher` in the topbar. Accessibility shape:

* Trigger: `aria-haspopup="dialog"` + `aria-expanded`.
* Popover: `role="dialog"` with `aria-label` from
  `t("display.button")`.
* Sections: `role="group"` + `aria-label`, segmented buttons
  with `aria-pressed`.
* Esc closes the popover and restores focus to the trigger.
* Outside-click closes without focus return.

### "How it works" — `/pipeline`

`frontend/src/features/pipeline/PipelinePage.tsx`. A static seven-
step grid (Camera → Capture → Detection → Identification →
Attendance → Policy → Report) using the design's existing
`.pipeline-big` + `.pb-step` classes from
`styles-enhancements2.css`. Numbered tile (`pb-num`), accent-
soft icon tile (`pb-icon`), italic display title (`pb-title`),
prose body (`pb-text`), and a mono `step N / 7` footer
(`pb-meta`). Visible to every authenticated role; the
placeholder route in `App.tsx` was replaced with a real
`<PipelinePage>` element.

### "API Reference" — `/api-docs`

`frontend/src/features/api-docs/ApiDocsPage.tsx`. Two stacked
cards: an operator overview (auth, tenant cookie, roles, rate
limits) and an iframe embed of `/api/docs` (FastAPI's Swagger
UI). The iframe is same-origin via the Vite proxy, so the
operator's session cookie flows through and "Try it out" works
without a separate auth dance. An "Open in new tab" link
escapes for full-screen use.

The route is gated by an inline `<AdminOnly>` wrapper in
`App.tsx` (redirects non-Admins to `/dashboard`). The backend
remains the source of truth — `/api/docs` itself doesn't refuse
non-Admins because Swagger is a useful internal-tooling
surface; the gate is a UX guard so the sidebar stays coherent.

### Accessibility sweep

* `:focus-visible` outlines added to a global rule at the
  bottom of `styles-enhancements3.css` — 2px accent outline +
  2px offset, with a dark-theme override that uses
  `--accent-text` so the halo stays readable on the deep
  background. Pointer users see no change; keyboard users see
  the affordance.
* `aria-label` on every icon-only button across the topbar
  (NotificationBell, DisplaySwitcher, LanguageSwitcher).
* `aria-haspopup` + `aria-expanded` on dropdown triggers.
* `role="dialog"` + `aria-label` on popover panels.
* `aria-pressed` on the segmented Theme + Density buttons.
* `frontend/CLAUDE.md` documents the rule going forward — every
  new interactive element must ship with an accessible name.

## Notes for whoever resumes next

* **M2 core is now complete.** Tag `v1.0-m2` after this commit
  per the prompt.
* **Branding accent + dark theme.** The dark palette inherits
  the per-tenant accent via OKLCH overrides already declared in
  `[data-theme="dark"]`; if Branding (P4) adds a new accent
  preset, smoke-test it under both themes before shipping.
  The accent token is the only colour the operator can change,
  and it's the only one that bridges both palettes — keep it
  curated (no free-form hex; the BRD red line still holds).
* **`prefers-color-scheme` watcher.** The watcher only fires
  while `theme === "system"`. If a user picks `light` or `dark`
  explicitly, OS changes don't override their choice — that's
  the point. If you ever want a "follow OS at all times" mode,
  add it as a fourth value rather than reinterpreting `system`.
* **API docs role gate.** `<AdminOnly>` is a UX guard, not a
  security boundary. The backend serves `/api/docs` and
  `/api/openapi.json` to anyone authenticated because they're
  useful for internal tooling and integration testing. If the
  BRD ever requires a hard 403 for non-Admins on the spec, gate
  the FastAPI endpoint itself — the iframe embed will then 403
  too.
* **Density × custom-css surfaces.** Most components inherit
  the `--density` multiplier through padding rules in
  `styles.css`. Pages that use raw pixel padding inside inline
  `style` props won't respond to the toggle; if a future page
  feels "stuck" under Compact, audit its inline styles and
  either move them to a class or multiply by
  `var(--density)`.
* **Pipeline page — design source.** The component reuses the
  design archive's class names without re-importing the JSX
  reference. Future enhancements (animations, illustrations,
  links to the relevant CMS page) should land in this file
  directly — it's not regenerated.
* **a11y backlog.** Focus outlines + aria-labels on the new
  surfaces are the floor, not the ceiling. The full a11y pass
  (skip-to-content link, landmark roles on the shell, keyboard
  nav for the sidebar) is M3 hardening territory and lives in
  the next phase plan.
