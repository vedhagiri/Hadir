## Goal

Maugood writes a daily attendance file the client ERP polls. Operators
configure format (CSV / JSON), an output subpath under the tenant
root, and a cron schedule in **Settings → Integrations → ERP
Export**. The runner shares APScheduler with P18 (one 60-second
poll across all scheduled work).

## What was done

### Migration 0020

`alembic/versions/0020_erp_export_config.py`. One per-tenant table:

| Table | Notes |
| --- | --- |
| `erp_export_config` (tenant_id PK) | enabled, format ('csv'\|'json'), output_path, schedule_cron, window_days (default 1), bookkeeping (last_run_at, last_run_status, last_run_path, last_run_error, next_run_at), updated_at. CHECK on format. Empty row seeded on the migrating tenant + provisioning. |

### Config knob

`backend/maugood/config.py` adds `erp_export_root` (default
`/data/erp`). Every operator-supplied path is joined under
`{root}/{tenant_id}/` before being written; nothing escapes that
prefix.

### Path resolver — `maugood/erp_export/paths.py`

The load-bearing P19 red line lives here. `resolve_safe_dir(...)`:

* Empty input → tenant root.
* Relative input → `tenant_root / raw`, resolved.
* Absolute input → resolved as-is.

After `Path.resolve(strict=False)`, the function calls
`Path.relative_to(tenant_root)` and raises `UnsafeOutputPath` on
any escape. Both PATCH and the runner re-validate (defence in
depth) so a manual SQL update can't bypass the check.

### Builder — `maugood/erp_export/builder.py`

* `fetch_rows(...)` runs the same join the Excel/PDF reports use,
  ordered by `(date asc, employee_code asc)`.
* `_status_for(...)` reduces the engine flags to a single string —
  `leave > absent > late > short > early_out > on_time`. Documented
  in the schema doc.
* `render_csv(...)` writes UTF-8, LF-terminated, no BOM, lowercase
  `true`/`false`, empty cells for None.
* `render_json(...)` produces `{metadata: {...}, records: [...]}`
  with a `schema_version` field — bumped on breaking changes.
* `filename_for(...)` enforces the spec'd
  `maugood-attendance-{YYYYMMDD}-{HHmmss}.{csv|json}`.

### Runner — `maugood/erp_export/runner.py`

* `run_export_now(scope)` — full pipeline: read config, resolve
  safe directory, build rows, render bytes, `write_bytes` (one
  call so the file appears whole), update `last_run_*` and
  `next_run_at` even on failure so the runner doesn't get stuck.
* `tick_due_exports()` — invoked by
  `maugood.scheduled_reports.runner._tick` so we don't run two
  APScheduler instances.
* `tenant_context("public")` for the cross-tenant registry scan,
  then `tenant_context(slug)` per tenant.

### Endpoints — `maugood/erp_export/router.py`

| Method + Path | Role | Notes |
| --- | --- | --- |
| `GET /api/erp-export-config` | Admin | Returns the config + `tenant_root` so the UI can show "files will land in …". |
| `PATCH /api/erp-export-config` | Admin | Validates the path on save (400 on traversal) and the cron via croniter (422 on bad expression). |
| `POST /api/erp-export-config/run-now` | Admin | Builds + writes the file under the tenant root **and** streams the same bytes back so the operator can verify locally. Audit row records run filename + range + row count. |

### Schema doc — `docs/erp-file-drop-schema.md`

Hand-this-to-the-ERP-team document. Covers semantics per column,
the `status` reduction, CSV layout (with example), JSON layout
(with example), versioning policy, atomicity / retry / window
behaviour, and the audit-trail expectation.

### Frontend — `frontend/src/erp-export/`

* `ErpExportPage.tsx` — Settings → Integrations → ERP Export. Form
  for enabled/format/output_path/cron/window_days, "Save changes"
  + "Run now" buttons. Run-now downloads the produced file via
  blob + anchor for verification, and the page surfaces the last-
  run status (path, error, next-run timestamp).
* Cron preview reuses the helper from P18.
* Wired into `SettingsTabs` ("ERP export").

## Notes for whoever resumes next

- **Atomic writes for big files.** `Path.write_bytes` is fine at
  pilot scale (single-day reports). When window_days grows past a
  month and the JSON payload tops a few hundred MB, switch to a
  ``tmp+rename`` pattern so the ERP polling watcher doesn't pick
  up half-written bytes.
- **Other report types.** The schedule has `report_type` baked in
  but the only allowed value is `attendance`. Adding camera-logs
  or audit-log exports here is a small repeat — extend the runner
  to dispatch on `report_type` and add the new column to
  `docs/erp-file-drop-schema.md`.
- **No retry queue.** A failed run leaves `last_run_status=failed`
  + an `last_run_error` message; the next cron tick will retry.
  If a tenant ever has a transient FS error pattern (full disk),
  add a P25-style "retry once after backoff" — for now operator
  visibility via the Settings page is enough.
- **Path-traversal regression coverage.** Both API PATCH **and**
  the runner re-validate via `resolve_safe_dir`. A future
  migration that adds pre-existing rows still has to clear the
  resolver before any disk write — keep the runner check even if
  the API gets a stricter Pydantic validator.
- **`.test` TLD gotcha.** Same as P11–P18 — the smoke uses
  `admin@pilot.maugood`.
