# P17 — PDF reports

**Phase:** v1.0 P17 (M2)
**Date (UTC):** 2026-04-25
**Engineer:** Suresh (MTS) via Claude Code
**Plan reference:** `v1.0-phase-plan.md §"P17 — PDF reports"`

## Goal

A branded, printable attendance report alongside the existing
Excel export. Same filter form drives both; the PDF carries the
tenant's letterhead (logo + name in the tenant's primary colour
and curated font), a summary block, and one section per employee
with daily rows + totals. Multi-employee reports page-break
between employees so HR can hand a single block to the right
person.

## What was done

### System deps

WeasyPrint pulls `pango`, `cairo`, and `gdk-pixbuf` via cffi at
import time, so they have to land in the image before the package
can even load. `backend/Dockerfile` adds:

```
libpango-1.0-0 libpangoft2-1.0-0 libcairo2 libgdk-pixbuf-2.0-0 \
libffi-dev shared-mime-info fonts-liberation
```

`fonts-liberation` is the generic sans/serif fallback so PDFs
render on a host with no other fonts installed (the curated
families from P4 are CSS hints; WeasyPrint substitutes when the
exact font isn't on disk).

### Python deps

`backend/pyproject.toml` adds `weasyprint==62.3` and
`jinja2==3.1.4`. WeasyPrint 62.3 requires `pydyf<0.11` (upstream
issue #2129 — a `super().transform()` call broke when pydyf
shipped 0.11). We pin `pydyf==0.10.0` explicitly.

### Template

`backend/maugood/reporting/templates/attendance.html` — a single-file
Jinja template. Inline `<style>` (no external CSS lookups). Layout:

* `@page` rule with `@bottom-left { content: "Generated …" }` and
  `@bottom-right { content: "Page x of y" }` driven by CSS
  counters, so the footer renders on every page including those
  inserted by the page-break-between rule.
* Letterhead — tenant logo (data-URL embedded; falls back to the
  tenant initial in an accent-soft tile when no logo) + tenant
  name in the accent colour + meta block on the right (date range,
  generated-at, generated-by email).
* Summary block — date range, employee count, total rows, late
  count, overtime hours, optional department label.
* Per-employee section — accent-coloured header (name + code +
  totals), then a table of daily rows (date, in, out, hours,
  late/early/short/absent flags, OT minutes, policy).
* `page-break-before: always` on every employee after the first
  **only when the report covers more than one employee** — single-
  employee reports stay on one page.

### PDF builder

`backend/maugood/reporting/pdf.py`. Pure-ish module:

* `_query_rows(...)` — same SELECT-shape as the Excel builder but
  ordered by `(employee_code, date)` so the per-employee grouping
  reads top-to-bottom.
* `_build_employees(...)` — groups flat rows by employee_id and
  computes per-employee totals (days / late / early_out /
  short_hours / absent / overtime_minutes + overtime_hours).
* `_branding_for_tenant(...)` — reads `tenant_branding` (lazy-
  creates if absent), maps `primary_color_key` to a stable hex
  pair via the new `HEX_PALETTE` (deliberate, parallel to the
  OKLCH set used by the React shell — keeps the rendered colour
  predictable across PDF readers), and inlines the logo as a
  base64 `data:` URL so the renderer never opens a network socket.
* `build_pdf(...)` — renders the template via Jinja, then
  `weasyprint.HTML(string=html_str).write_pdf()`. Returns
  `(bytes, row_count)`.
* `filename_for(...)` — implements the spec'd filename:
  ``maugood-attendance-{tenant_slug}-{from}-to-{to}.pdf``.

The Jinja template is loaded with `select_autoescape(("html",
"xml"))`, so the `font-family` value (a curated CSS stack from
`FONT_OPTIONS`) needs `| safe` to keep its single quotes intact.
That's the only filter override; everything else renders escaped.

### Endpoint

`POST /api/reports/attendance.pdf` mirrors the existing
`/attendance.xlsx` endpoint:

* Same `AttendanceReportRequest` body (start, end, optional
  department_id + employee_id, max_days clamp).
* Same role gates — Admin / HR / Manager only; Manager auto-scopes
  to assigned departments; cross-department filter returns 403.
* Same date guards — start ≤ end, range ≤ max_days.
* Audit row written via `report.generated` with `format=pdf`
  alongside the existing fields.
* Empty-result manager-only path returns a branded "no data" PDF
  rather than a 4xx, matching the XLSX UX contract.

### Frontend

`frontend/src/features/reports/ReportsPage.tsx` gains a "Generate
PDF" button alongside the existing "Generate Excel". The same
filter inputs drive both — the click handler picks the endpoint
based on which button fired. Loading state is per-format so the
two buttons disable independently. `Content-Disposition` filename
is honoured for the download; default filename falls back to
`attendance_{from}_to_{to}.pdf` when the header is missing.

The page subtitle was updated to drop the "scheduled delivery is
deferred to v1.0" line (we're in v1.0 — and P18 is "scheduled
reports + email"). The help blurb explains both formats and the
manager scoping in plain English.

## Notes for whoever resumes next

- **Scheduled delivery (P18)** is the natural follow-up. The PDF
  builder already returns `(bytes, row_count)` and reads tenant
  branding without any HTTP context, so the scheduler can call
  `build_pdf(...)` directly inside a `tenant_context(...)` block
  and email the result.
- **Hex palette parallels the OKLCH set.** The React shell drives
  the live UI via OKLCH (P4 §"Curated palette"), but PDF
  readers handle OKLCH inconsistently. `HEX_PALETTE` in
  `maugood/reporting/pdf.py` keeps a deliberate hex pair per palette
  key — extend it whenever a new palette key lands. If the two
  sets ever drift, the live UI and the printed PDF will look
  noticeably different and HR will notice immediately.
- **Logo embedding.** Logos are read from disk and base64-encoded
  inline. PNG works directly; SVG works because WeasyPrint
  natively renders SVG. If a tenant uploads a JPEG via the
  branding admin (currently rejected by the magic-byte check), the
  PDF would still render the fallback initial — file an issue.
- **Multi-page footer.** `@page { @bottom-right }` uses CSS
  counters; if a future template change adds a custom front
  matter ToC, suppress the footer there with `@page :first { …
  none }`.
- **WeasyPrint 62 + pydyf 0.10.0 pin.** WeasyPrint 63 (when it
  ships) is expected to update the pydyf pin upstream. Re-evaluate
  the pin together with a WeasyPrint upgrade — don't bump pydyf
  alone.
- **`.test` TLD gotcha.** Same as P11–P16 — Pydantic `EmailStr`
  rejects reserved TLDs, so the smoke uses `admin@pilot.maugood`.
