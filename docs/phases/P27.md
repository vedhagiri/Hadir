### Code fixes

* **`backend/scripts/seed_admin.py`** + **`provision_tenant.py`** —
  added a 12-character minimum-length check. NIST SP
  800-63B § 5.1.1.2 floor + a comfortable margin. Operators
  trying to set a shorter password get a non-zero exit and
  a readable error. Closes **P27-007**.
* **`backend/hadir/cameras/rtsp.py`** — `parse_rtsp_url`
  scheme allowlist tightened from `(rtsp, rtsps, http,
  https)` to `(rtsp, rtsps)`. The `http`/`https` allowance
  was an SSRF surface against internal HTTP services
  (cloud metadata, localhost ports, …) reachable from the
  backend. Operators with HTTP-MJPEG cameras now bridge
  through an RTSP proxy. Closes **P27-008**.

### Manual review

Walked all ten categories from the prompt. Each section in
`docs/security-review.md` records the question, what was
checked, and the result.

Highlights:

* **Session fixation** — safe by design.
  `auth.sessions.create_session` always allocates a fresh
  `secrets.token_urlsafe(48)`; we never accept a
  client-supplied id.
* **Brute-force** — protected via P3 rate limiter.
* **OIDC state + nonce + JWKS validation** — wired through
  authlib + the signed cookie carrying state+nonce.
* **Authorization** — counted route decorators vs guard
  references in every router file. Every authenticated
  endpoint takes `CurrentUser` (or a subclass via
  `require_role`/`require_super_admin`); the only
  intentional anonymous endpoints are `auth/login` and the
  OIDC callback.
* **IDOR** — every photo + request lookup filters on
  `tenant_id` AND the inner foreign-key id. Cross-tenant
  reads return 404 by design.
* **SQL injection** — `?q=' OR 1=1--` and
  `?q=%; DROP TABLE employees;--` against `/api/employees`
  both returned an empty 200. Parameterised queries hold.
* **XSS / CSS injection** — branding's `<style>` tag reads
  values from the server-curated palette (8 OKLCH options
  + 3 font stacks) — user submits a key, server resolves
  the CSS string. No operator-supplied CSS lands in the
  DOM. No `dangerouslySetInnerHTML` outside the comment in
  `BrandingProvider.tsx`.
* **CSRF** — `SameSite=Lax` on every cookie + CORS
  allow-list (P23 startup check refuses production boot
  without it) + JSON-only API. Three layers; no token
  needed.
* **Crypto rotation** — annual + on-incident, documented
  in `docs/disaster-recovery.md`.
* **Secrets in repo / git history** — clean.
  `git ls-files | xargs grep` and `git log -p -G ...` for
  PEM headers and 12-char password literals matched only
  `.env.example` placeholder text and synthetic test
  passwords.
* **Tenant isolation** — P1 canary + P5 two-tenant smoke
  re-run, **16/16 pass**.
* **File upload** — every path uses magic-byte validation;
  filenames are server-generated UUIDs;
  `HADIR_REQUEST_ATTACHMENT_MAX_MB` enforced server-side.
* **Audit log immutability** — `hadir_app` rejected on
  DELETE / UPDATE / TRUNCATE of `main.audit_log` (verified
  this session).

## Backward compatibility

* All 435 prior tests pass on the bumped dependency set.
* No new tests added — security review is audit-driven.
* CLI scripts now refuse short passwords (12-char floor).
  Operators with shorter pilot passwords need to re-seed
  (the seed script is idempotent — running again with a
  conformant password just upserts the row).
* `parse_rtsp_url` rejects http/https. Existing RTSP/RTSPS
  cameras unaffected; an operator who configured an HTTP
  MJPEG camera before this phase has to reconfigure.
* Image bumps are transparent — same nginx config + same
  supercronic crontab.

## Notes for whoever resumes next

* **`docs/security-review.md` is append-only.** Future
  reviews add a new top-level section; never edit a past
  finding.
* **Dependency-refresh cadence is monthly.** Set a
  calendar reminder. The M3 backlog tracks the four
  deferred items; the next refresh closes them and updates
  this doc with the new scan results.
* **Password reset endpoint** — there isn't one. The pilot
  uses operator-driven seeding + OIDC. v1.x adds a
  self-service reset flow, which will need its own
  password-policy hook (probably re-using
  `_enforce_min_length` from `provision_tenant.py`).
* **HTTP-MJPEG cameras** — if Omran's IT comes back with a
  legitimate use case, the right answer is a
  per-deployment RTSP-proxy sidecar (e.g.,
  `ffmpeg`/`mediamtx`) rather than reopening the http(s)
  scheme allowlist. The SSRF surface isn't worth it.
* **Open critical item carries forward**: Omran HR
  native-speaker review of the Arabic translations from
  P21 is still pending before v1.0 launch.
